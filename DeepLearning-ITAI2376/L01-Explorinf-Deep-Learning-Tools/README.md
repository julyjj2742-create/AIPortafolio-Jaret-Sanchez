# L01 - Exploring Deep Learning Tools: A No-Code Introduction to TensorFlow and Keras

## Overview
In this lab, I worked with a pre‑built deep learning model (VGG16) to get familiar with the basic workflow of a deep learning project. I didn’t have to write any model architecture myself — instead, I focused on understanding how data is preprocessed, how the model is structured, and how predictions are generated.

## Purpose
The purpose of this lab was to help me understand the overall flow of a deep learning project without getting overwhelmed by code. The lab walked me through loading the VGG16 model, viewing its architecture, preprocessing images, and using the model to classify them. I also used widgets to upload my own images and see the model’s predictions in real time.

## Key Concepts

### Using Pretrained Model
I learned how to load and use the VGG16 model, which is already trained on ImageNet. This helped me understand how deep learning models can be reused without training them from scratch.

### How Image Preprocessing Works
Before making predictions, the image has to be resized, converted to an array, and preprocessed. The lab showed me how important these steps are for getting correct results.

### Making Predictions
I learned how to run an image through the model and decode the top predictions so I could see what the model thinks the image contains.

### Testing With My Own Images
The upload widget let me try different images and see how the model responded. This made the lab more interactive and helped me see how the model behaves with real examples.
